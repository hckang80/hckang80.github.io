바람직한 CSS 작성 방법(지극히 주관적)

2018-01-25
CSS
CSS, selector

(참고 : 글쓴이의 주관적인 해석에 의해 작성된 것이며, 글 특성상 경어는 생략하도록 하겠습니다)

요즘은 Sass, Less 등 전처리기의 도움으로 자칫 복잡해 질 수 있는 css 코드도 효과적인 관리가 가능하지만 바람직한 css를 작성하는 기본은 잊지 않아야 한다.
그럼 글쓴이의 경험에 의한 바람직한 css 작성 방법에 대해 알아보겠다.

내용이 중복되지 않도록 한다.

위와 같은 경우, 배경이미지 등 여러 내용이 동일함에도 불구하고 중복되어 사용되고 있다. 이중 클래스 등을 두어 공통 내용은 모아서 작성하고 값이 다른 배경이미지 위치만 각기 넣는 형태가 바람직하다.

선택자를 너무 간결하거나 복잡하게 사용하지 않는다.

선택자가 간결할 경우, 웹 사이트가 확장 됨에 따라 중첩된 선택이 일어나 의도하지 않은 변형이 일어날 확률이 현저히 많아진다. 반대로 너무 길어지게 되면 가독성이 안좋아지고, 코드의 일관성을 맞추기도 어려워진다.
위와 같은 경우, 각 요소의 첫 번째 선택자를 두 단어 이상 조합된 형태로 작성만 하여도 충분하다.

불필요한 사이즈값은 주지 않는다.

요소의 성질을 잘 파악하지 못하는 경우 이러한 경우가 많이 발생한다. 블록요소로써 x축으로 이미 꽉 차는 형태인데도 불구하고 습관처럼 width:100%을 추가로 작성하는 경우가 있다. 또한 float 이나 position 이후 부모 요소의 높이가 잡히지 않는 이유로 height 속성을 남발하기도 한다. 이는 요소의 높이를 고정하게 되므로 문제가 될 소지가 다분하다.
